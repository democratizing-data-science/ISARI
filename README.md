# AERA ISARI Workshop Dashboard

The **AERA ISARI Workshop Dashboard** is an interactive, single-file HTML dashboard for teaching, documenting, and integrating AI-assisted qualitative and mixed-methods research workflows. The dashboard brings together methodological frameworks, local AI reasoning, privacy-conscious analytic practices, and visual explanations for a suite of qualitative data science tools.

## Overview

The workshop dashboard is designed as a public-facing educational and methodological resource for researchers, instructors, students, and research teams working with qualitative evidence, mixed-methods designs, visual analytics, and AI-assisted interpretive workflows.

The dashboard emphasizes **Truly Equal Status Design (TESD)**: computational, visual, quantitative, qualitative, spatial, temporal, relational, affective, and multimodal evidence should support one another without allowing automated outputs to replace researcher judgment, participant context, or interpretive accountability.

In this framework, computational outputs are treated as analytic waypoints, not findings. Researchers are expected to return to original evidence before making claims, writing interpretations, or using AI-generated text in scholarly work.

**Book Description:**

- Related frameworks such as NAQD, GRATIS, and VERITAS for network, temporal, and interactional evidence.

The book and dashboard share the same methodological commitment: computational tools can structure, visualize, classify, transcribe, or summarize evidence, but interpretation remains the responsibility of the researcher.

# ISARI Help and Troubleshooting Dashboard

The **ISARI Help and Troubleshooting Dashboard** is a companion single-file HTML dashboard focused on helping users run, repair, troubleshoot, distribute, and maintain ISARI on macOS and Windows.

It is intended for workshop participants, support staff, instructors, and users who download, unzip, AirDrop, move, copy, or redistribute ISARI and then encounter macOS launch problems.

## What the Help Dashboard Covers

The help dashboard includes tabs for:

- **Start Here** — Universal post-transfer repair kit for ISARI packages moved by AirDrop, ZIP, OneDrive, browser download, external drive, or Windows-to-macOS transfer.

- **Symptoms** — Common macOS symptoms such as `permission denied`, `operation not permitted`, Gatekeeper warnings, Python library verification messages, broken launchers, and missing `ISARI_AI/ISARI.command` messages.

- **Why It Happens** — Explanation of Unix-style executable permissions, macOS quarantine attributes, portable Python links, and why transfers can break launch behavior.

- **Folder Layout** — Required and recommended ISARI folder structures, including the rule that `ISARI.app` and `ISARI_AI` must remain sibling items in the same parent folder.

- **Quick Diagnosis** — Checks for determining whether the internal ISARI runtime works and whether the issue is likely the app wrapper, permissions, quarantine, Python links, or folder layout.

- **Short Repair** — A shorter repair option for users who are already inside the correct ISARI parent folder.

- **App Wrapper Repair** — Instructions for rebuilding the outer `ISARI.app` wrapper when the internal launcher works but the app wrapper does not.

- **Hide / Unhide** — macOS and Windows commands for hiding or unhiding the `ISARI_AI` folder and keeping the user-facing folder clean.

- **Health Checks** — Browser and Terminal checks for confirming whether ISARI is running or stopped, including the local health endpoint.

- **Distribution** — Recommendations for distributing ISARI, including the use of `.tar.gz` archives on macOS to better preserve executable permissions.

- **Summary** — A concise support pathway: keep layout intact, run repair after transfer, test the internal launcher, and rebuild the app wrapper only if needed.

- **About ISARI and Contact** — High-level description of ISARI, how to use the help dashboard, official access links, Code Ocean citation, and developer contact information.

---

## Key Features of the Help Dashboard

- **Copy-ready repair scripts**  
  Includes scripts users can copy directly into Terminal.

- **Auto-detecting repair workflow**  
  The main repair script searches for the correct ISARI folder and does not require users to manually hard-code the folder path.

- **macOS quarantine repair**  
  Removes `com.apple.quarantine` attributes that may trigger warnings such as Apple being unable to verify a file.

- **Executable-permission repair**  
  Restores executable permissions for `.command` files and app-wrapper files inside `ISARI.app`.

- **Portable Python repair**  
  Repairs `python3` and `python` links when portable Python symlinks or wrappers are damaged during transfer.

- **Layout verification**  
  Checks whether `ISARI.app` and `ISARI_AI` are correctly located next to each other.

- **App-wrapper repair**  
  Provides a rebuild path for `ISARI.app` when `ISARI_AI/ISARI.command` works but the outer app wrapper fails.

- **Distribution guidance**  
  Explains why `.tar.gz` is recommended for macOS distribution and why ZIP, browser downloads, AirDrop, OneDrive, and cross-platform transfers may require repair.

# Ethical Use

These dashboards treat AI as a **support tool**, not as a replacement for researcher interpretation.

Researchers should:

- return to original qualitative evidence before writing claims,
- distinguish descriptive patterns from interpretive conclusions,
- avoid uploading sensitive data to external systems unless approved,
- protect participant confidentiality and location privacy,
- document analytic decisions,
- treat AI-generated text as provisional brainstorming,
- verify all outputs against source evidence,
- treat transcripts, summaries, and visual descriptions as provisional until checked,
- preserve the researcher’s interpretive authority throughout the workflow.
